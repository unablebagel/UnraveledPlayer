UnraveledPlayer: from an attack sketch to SIEM-style alerts

This portfolio prototype supports research on augmenting SIEM operations with threat-model knowledge. A static threat model tells us the attack paths that are possible; the goal is to combine that knowledge with incoming alerts to infer an attacker’s intent, track compromised components, and anticipate the next likely target.

Why this tool exists

Real multi-attacker APT data is scarce. The Unraveled dataset provides a stand-in SIEM feed, but contains one campaign, so this editor lets me visually author synthetic, interleaved attacks. The diagram is the input: each move becomes time-ordered OCSF-style alerts that can exercise multi-attacker attribution and compromise-state tracking.

Try it in four steps

  1. Choose a topology at the top. Grey nodes have no pinned IP, so they cannot be used as move endpoints.
  2. Set up an attacker in the Attackers panel. Select its row to make it the current attacker; add another attacker to model concurrent activity.
  3. Draw moves on the diagram: click a source (or EXTERNAL for initial access), then click a destination. Repeat to build the attack path. Edit techniques, ports, and timestamps in the timeline; drag rows to interleave attackers.
  4. Compile & validate. Review the report, download the generated synthetic_alerts.jsonl, then switch to Inferred sessions to compare your authored attackers with the engine’s attribution.

Playback and useful controls

Use the slider or play button to replay the scenario in time order. Colored dots show attackers that have touched a node; click a legend chip to focus one attacker. Save/load preserves a scenario as JSON, while Import alerts reverses a previously generated JSONL file back into an editable scenario.
